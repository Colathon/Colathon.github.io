# 网络基础知识速查（日常用 + 挂代理够用版）

> 一句话主线：**数据从你设备的某个接口出发 → 经过路由器层层转发 → 到达目标的某个端口**，而 VPN / 代理无非是在这条路的起点动手脚。

---

## 一、地址：数据怎么找到你

### IP 地址
每台联网设备的"门牌号"。底层网络只认 IP，不认域名。

IPv4 总共约 **43 亿个**，早就不够用，靠两个办法解决：
1. **公网 IP + 内网 IP 分开**（见下）。
2. **IPv6**：地址长（`2001:0db8:...`），数量大到约 340 万亿亿亿个，目前与 IPv4 并存过渡中。

### 公网 IP vs 内网 IP（接力关系，不是二选一）

用"住公寓楼网购"理解：

| | 类比 | 作用 | 范围 |
|---|---|---|---|
| **公网 IP** | 整栋楼的街道地址（全城唯一） | 让**整个互联网**找到你家（你家路由器） | 全球唯一，运营商分配 |
| **内网 IP**（`192.168.x.x`） | 楼里的房间号（502 室） | 让**路由器**分清是家里哪台设备 | 只在你家局域网内有效 |

- 全世界无数家庭都在用 `192.168.x.x`，因为它出了门就失效，不冲突。
- 数据流程：手机请求 → 路由器记账"是 192.168.1.5 发的" → 用**公网 IP** 寄出去 → 网站只看得到你家公网 IP → 回复发回公网 IP → 路由器查账，准确送回手机。
- 路由器做这个翻译/记账的机制叫 **NAT（网络地址转换）**。
- **一栋楼（全家几十台设备）对外只占一个公网 IP**，这是 43 亿 IP 还能撑住的最大原因。

### 回环地址 `127.0.0.1`
含义是**"本机自己"**。数据发到这里**不出门**、不经过网卡和路由器，在机器内部转一圈就回来（所以叫"回环 / loopback"）。

- 每台设备上都叫 `127.0.0.1`，但**各指各的自己**（像每个人说"我自己"）。
- 手机用 `127.0.0.1` 永远连不到电脑。
- 配代理时的 `127.0.0.1:7890` = "本机自己的 7890 端口"，数据在本机内部从浏览器交给代理软件，全程没出门。

**三类地址别混：** 公网 IP（对外总门牌）、内网 IP（家里各设备房间号）、回环地址（每台设备的"我自己"）。

## 二、端口：找到机器后，找到哪个程序

- IP 找到机器，**端口**找到机器里具体哪个服务（"房间里的分机号"）。
- 通信**双方各有自己的端口**，每次连接是"我的某端口"对接"对方某端口"。
- **服务器端**：端口固定且公开，别人才找得到（80 网页、443 加密网页、53 DNS、22 远程登录）。
- **客户端（你）**：端口大多临时随机分配，用完回收。
- 代理软件（如 Clash）在**本机**开个端口（常见 `7890`）待命，你把流量指向 `127.0.0.1:7890` 即可。**这个端口是本地软件层面的，不是机场服务商给的。**

---

## 三、路由器：数据在路上怎么转发

- **路由器 ≠ WiFi。** 路由器负责"在不同网络间转发数据、决定走哪条路"；WiFi 只是一种无线连接方式（也可用网线，照样走路由器）。
- 家里那个盒子是"路由器 + 无线 AP + 交换机"打包在一起。
- **学校 WiFi**：也有路由器，规模大得多（企业级设备 + 多个 AP）。
- **用手机流量**：依然有路由器，只是在运营商机房里替你转发。
- **结论：只要数据要走出本地访问互联网，路上一定有一连串路由器接力转发**，你家那台只是第一站。（命令 `traceroute` 可看到数据路过了多少台中转设备。）

## 四、传输层：数据怎么搬运（TCP vs UDP）

| | TCP | UDP |
|---|---|---|
| 可靠性 | 可靠，先握手、逐包确认、丢了重发 | 不可靠，发出去不管，不确认不重发 |
| 速度 | 稳但相对慢 | 快但可能丢包 |
| 适用 | 网页、下载、收发消息（一个字都不能错） | 视频通话、游戏、直播（丢一点没关系，不能卡） |

> 后面代理协议的"快不快"，很大程度看它走 TCP 还是 UDP。

## 五、应用层：上网的完整流程（DNS / HTTP / HTTPS）

**顺序：DNS 找路 → IP 和路由送达 → HTTP 对话 → HTTPS 上锁。**

### DNS（域名系统）= 互联网通讯录
把 `google.com` 翻译成机器能用的 IP（`142.250.x.x`）。是**每次访问网站的第一步**：先翻译，再通信。

- **DNS 污染 / 劫持**：翻译环节被动手脚，返回错误 IP，导致打不开网站。问题**不在通信本身，在"翻译"被做了手脚**。
- 这就是代理 / VPN 工具里常有"DNS 设置"的原因——绕开被污染的翻译，去问干净可信的 DNS。

### HTTP = 网页传输的"对话规则"
浏览器发**请求（Request）**，服务器回**响应（Response）**。缺点：**明文传输**，中间任何经手环节都能看到你传的全部内容。

### HTTPS = 加了锁的 HTTP（S = Secure）
内容**全程加密**，中间截获也只是乱码。地址栏 🔒 图标表示连接已加密。输密码、付款务必确认有锁。

> **重要区分：** HTTPS 加密的是**通信内容**，但**不隐藏你访问了哪个网站**。"内容被保护"和"行踪被隐藏"是两码事，后者才是 VPN / 代理要解决的，两层可叠加。

## 六、VPN 与代理：在路的起点动手脚

| | VPN / TUN 模式 | 代理 / 端口模式 |
|---|---|---|
| 原理 | 新建**虚拟网络接口**（tun/tap），改路由规则，接管流量 | 本机开个端口（如 7890）待命 |
| 范围 | **全局**，所有程序流量被接管，程序自己不知道 | 只有**设置成走代理**的程序才走，更可控 |

- 像 Clash 这类工具**两种模式都能开**：只开本地端口 = 代理模式；开 **TUN 模式** = 变成 VPN 那种全局接管。
- **机场 / 节点 / 订阅**：机场 = 提供代理服务的服务商；节点 = 一个个可连服务器；订阅 = 打包一堆节点的链接，导入即用。

---

## 七、代理协议：本质都是"加密打包 + 伪装得不像代理"

区别只在手段：**拼速度 / 弱网** 还是 **拼隐蔽**。

| 协议 | 特点 |
|---|---|
| **Shadowsocks (SS)** | 老牌轻量，加密让流量不像代理 |
| **VMess / VLESS** | V2Ray 体系，VLESS 是简化升级版，更轻更快 |
| **Trojan** | 伪装成正常 HTTPS 流量，难区分 |
| **TUIC** | 基于 **QUIC**（跑在 UDP 上的新一代传输，HTTP/3 用它）。主打**快、抗丢包、弱网稳**，适合游戏 / UDP 转发 |
| **Hysteria** | 同样基于 QUIC，专为高丢包高延迟优化，多发冗余换速度，弱网很猛 |
| **AnyTLS** | 较新，把流量**伪装成普通 TLS（HTTPS）流量**，难被从正常上网里识别出来 |
| **Mieru** | 较新的混淆类协议，注重让流量特征不明显 |

> 日常不必懂内部实现，只要知道：**它们都为了又快又不容易被封地传输流量。连不上或慢，换一种协议的节点试试。**

## 八、看懂客户端界面（Clash 类）

### 顶部是"策略组"，不是节点
| 标签 | 含义 |
|---|---|
| `URLTest`（自动选择） | 自动测速，帮你挑最快的 |
| `Fallback`（故障转移） | 主节点挂了自动切备用 |
| `Selector` | 手动选择器，你点哪个用哪个 |

机场常按**用途**（ChatGPT 组）或**协议**（Tuic 组）帮你分好组。

### 节点标签
- `UDP`：支持 UDP 转发，游戏 / 语音 / 视频需要它。
- `Tuic` / `AnyTLS` / `Mieru` 等：标该节点用什么协议。

### 线路品质（不是协议）
- **专线**：优化过的专用线路，更稳更快延迟低。
- **全网优化**：普通公网做过路由优化，性价比线路，质量略逊专线。

## 九、延迟 vs 带宽：挑节点的核心（最容易踩的坑）

界面那个数字（ms）**只是延迟，没告诉你带宽**。

**高速公路比方：**
- **延迟（Latency）** = 车从这头到那头多快 → 反应快不快（单位 ms，越小越好）。
- **带宽（Bandwidth）** = 有多少条车道 → 一次能运多少（单位 Mbps）。
- 两者**互不决定**：可以车快但车道少，也可以车慢但车道多。

| 场景 | 看哪个 |
|---|---|
| 网页 / 聊天 / 游戏 / 语音 | **看延迟**，选数字小的 |
| 看视频 / 下载 / 大文件 | 延迟够用即可（200ms 内无所谓），**重点挑实际跑起来流畅的** |

### 延迟怎么测的
客户端请求一个空响应测试 URL（如 `gstatic.com/generate_204`），只看通不通、来回多快，不下载内容。测的是**你 → 节点 → 测试 URL 来回**这一段，核心反映**你到这台节点的连接质量**。换节点延迟会变，变的是"你到节点"这段。

### 延迟由什么决定
- **物理距离**（下限，光速限制，改不了）：东京 → 日本/港/台/新加坡低，→ 巴西/南非高。
- **网络路径质量**（可优化）：绕路、拥堵（高峰更慢）、运营商互联质量。专线就是买了优化过的直达路径。
- `Error` = 该节点当前连不上 / 测速失败（故障、被封、或路径不通）。

### 带宽为什么不显示
测带宽要真下载一大坨数据，费流量又给服务器添负担，几十个节点全测会把机场测崩。只能实际用时体感。带宽受**节点带宽上限、当前多少人挤、机场限速、路径最窄段**影响。

## 十、实用挑节点法

1. **大部分时候**：用 `自动选择`（URLTest），省心。
2. **看 ChatGPT / 特定服务**：用机场分好的对应组。
3. **手动挑**：选**延迟小、非 Error**，优先日/港/台/新加坡 + 标"专线"的。
4. **看视频卡但 ping 低**：几乎都是**带宽问题**，果断换节点（哪怕延迟数字大点）。
5. **某节点连不上 / 卡**：换节点，或换一种**协议**的节点（AnyTLS 不行试 Tuic）。
6. **晚高峰卡**：多半是节点被挤（带宽被分薄），换冷门或换地区。

> 核心思路：**延迟决定"反应快不快"，带宽决定"能不能扛大流量"。** 按用途分维度选，比记住任何协议名都管用。
